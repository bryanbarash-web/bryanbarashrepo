The Dutchie Claude plugins marketplace allows installing plugins with MCP servers, hooks, and skills. A compromised or malicious plugin could inject instructions, intercept data, or establish persistence via hooks that execute on every tool call.

No code signing for plugins. No runtime sandboxing of MCP servers. Edge channel plugins may bypass thorough review. No automated security scanning of plugin code.

Intellectual Property Leakage to Model Provider

● Partial
Anthropic's data usage policy (no training on API data). Enterprise agreements likely in place. VAPE environments are ephemeral and isolated.
No on-premise model option in production use. No content filtering for outbound API calls. No classification system to prevent highly sensitive code from being sent.

🚀 Top 5 Recommendations

1
Implement AI-Specific DLP (Data Loss Prevention) Layer
Addresses: Tilda sensitive data exposure, Snowflake mass PII extraction, Datadog log exfiltration

Deploy a DLP proxy layer between AI tools and data sources that classifies and redacts sensitive data (PII, PHI, financial records, cannabis purchase history) before it reaches model providers or is displayed in chat interfaces. This is the single highest-impact control given Dutchie's position in a regulated industry where customer purchase data carries elevated compliance obligations. Implement query result size limits, PII detection on outputs, and alerting on bulk extraction patterns. Priority targets: Snowflake query results, Datadog log searches, and Tilda Slack responses.
Urgent
2
Enforce Read-Only Defaults with Explicit Write Escalation
Addresses: Cloudflare infrastructure modification, Retool privilege escalation, Datadog monitor deletion, accidental destructive operations

All MCP servers and AI tool integrations should default to read-only access. Write operations (creating monitors, modifying DNS, granting admin roles, updating WAF rules) should require an explicit escalation step: either a separate authentication challenge, a mandatory approval from a second team member, or a time-limited write token. This follows the principle of least privilege and dramatically reduces the blast radius of prompt injection attacks, model hallucinations, or user error. Implement this first for Cloudflare MCP and Retool DB, then extend to Datadog.
Urgent
3
Centralized AI Audit Log & Anomaly Detection
Addresses: Tilda cross-surface audit gaps, attribution gaps, insider threat detection, compliance reporting

Build a unified audit log that aggregates all AI tool interactions across every form factor: Tilda (Slack, web, MCP), Claude Code (CLI, IDE), Cursor, VAPE/BildA, and all MCP servers. Each log entry should capture: user identity, tool invoked, query/action performed, data accessed, and timestamp. Feed this into Datadog with anomaly detection monitors that alert on: unusual query volumes, after-hours access patterns, bulk data retrieval, privilege escalation attempts, and cross-system action chains. This is essential for incident response and regulatory compliance in the cannabis industry.
High
4
Plugin Security Hardening & Supply Chain Controls
Addresses: Plugin supply chain risk, MCP server compromise, hook-based persistence, configuration drift

Harden the Dutchie Claude plugins marketplace with: (a) code signing for all published plugins using a Dutchie-controlled signing key, (b) automated security scanning (SAST/SBom) in the plugin validation CI workflow, (c) runtime sandboxing for MCP servers (restrict network access, filesystem access, and available system calls), (d) mandatory security review for any plugin that registers hooks (which execute on every tool call), and (e) enforced plugin version management with automatic alerts when developers run outdated versions. Move the edge channel behind an explicit security review gate.
High
5
Per-User Credential Scoping & Token Rotation
Addresses: Datadog shared credentials, VAPE default database passwords, GitHub token scope, Cloudflare single OAuth token

Replace shared server-side credentials with per-user scoped tokens wherever possible. For the Datadog MCP, implement per-user API keys or a token-brokering service that issues short-lived, scoped tokens. Rotate VAPE sidecar database credentials per-session instead of using static defaults (postgres/postgres, sa/LLX-admin). Scope GitHub CLI tokens to the minimum required permissions for each user's role. For Cloudflare, implement token scoping by zone/service rather than granting full API access. This ensures that compromising one user's session doesn't grant access to all systems, and enables precise revocation during incidents.
High
